域名与 DNS

1 购买域名

在阿里云、GoDaddy 或 Namecheap 购买 aquavir-kb.org（或自定义域名）
如已购买，跳过此步骤。

2 添加 DNS A 记录

在域名管理后台的 DNS 设置中添加两条 A 记录：

主机记录记录类型记录值
@A服务器公网 IP
wwwA服务器公网 IP

3 验证 DNS 生效

DNS 全球生效需 5-30 分钟。在命令行测试：

nslookup aquavir-kb.org
# 应返回你的服务器公网 IP

服务器基础配置

1 SSH 登录

在本地终端执行，将 <IP> 换成你的服务器公网 IP：

ssh root@<你的服务器IP>

首次登录会提示确认指纹，输入 yes 回车。密码在云服务器控制台的「重置密码」中设置。

2 安装 Docker

curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

3 验证安装

docker --version
# Docker version 26.x.x ...

docker compose version
# Docker Compose version v2.x.x ...

上传代码

启动服务

1 启动所有容器

docker compose up -d

首次启动会拉取镜像（postgres:16、nginx:alpine）并构建 API 镜像，约需 3-5 分钟。

2 查看启动状态

docker compose ps
# 应显示 3 个服务都是 Up 状态:
#   db      Up (healthy)
#   api     Up
#   nginx   Up

docker compose logs -f
# 实时查看日志，Ctrl+C 退出

常见问题：nginx 启动失败

如果 SSL 证书还没申请，nginx 会因找不到证书文件而失败。先注释掉 docker-compose.yml 中 nginx 的 443 端口映射和证书卷，或先完成下一步再启动 nginx：

docker compose up -d db api
# 等证书申请完后:
docker compose up -d

配置 HTTPS

1 安装 Certbot

apt update
apt install -y certbot

2 申请免费 SSL 证书

先确保 80 端口可访问（DNS 已生效）：

# 暂时停止 nginx 释放 80 端口
docker compose stop nginx

# 用 certbot 独立模式验证域名
certbot certonly --standalone \
  -d aquavir-kb.org \
  -d www.aquavir-kb.org

# 成功后会显示:
# Certificate is saved at: /etc/letsencrypt/live/aquavir-kb.org/fullchain.pem

3 重启所有服务

docker compose up -d
# 现在 nginx 看到证书存在，会正常启动

4 设置自动续期

Let's Encrypt 证书有效期 90 天，需要自动续期：

crontab -e
# 添加一行：
0 3 1 * * certbot renew --quiet --pre-hook "docker compose -f /opt/deploy/docker-compose.yml stop nginx" --post-hook "docker compose -f /opt/deploy/docker-compose.yml start nginx"

请求路由逻辑

/                     → nginx 直接返回静态 index.html（极快）
/static/vendor/*      → nginx 静态文件，1 年缓存
/downloads/*          → nginx 静态文件 + 目录浏览
/api/*                → proxy 到 FastAPI（限流 15r/s）
/docs, /redoc         → proxy 到 FastAPI
/viruses, /search ... → try_files 静态 → @api 代理 SSR Jinja2

日常运维

常用命令

# 查看服务状态
docker compose -f /opt/deploy/docker-compose.yml ps

# 查看实时日志
docker compose -f /opt/deploy/docker-compose.yml logs -f api

# 重启 API 服务
docker compose -f /opt/deploy/docker-compose.yml restart api

# 更新代码后重新构建
docker compose -f /opt/deploy/docker-compose.yml build api
docker compose -f /opt/deploy/docker-compose.yml up -d api

# 备份数据库
docker compose -f /opt/deploy/docker-compose.yml exec db \
  pg_dump -U aquavir aquavir_kb > backup_$(date +%Y%m%d).sql

# 全部停止（不删除数据卷）
docker compose -f /opt/deploy/docker-compose.yml down

# 全部停止并清除数据（危险！）
docker compose -f /opt/deploy/docker-compose.yml down -v

监控建议

至少每周检查一次

  • 磁盘使用：df -h（50GB 通常够用，但 PG 会随时间增长）
  • 服务状态：docker compose ps
  • SSL 证书有效期：certbot certificates
  • API 健康检查：curl -s https://aquavir-kb.org/api/health
