# CrustaVirus DB 组会汇报稿

生成日期：2026-05-13  
对应前端页：`docs/group_meeting_progress_20260513.html`

## 当前进展

大家好，我今天汇报一下甲壳动物病毒数据库的最新进展。刚刚重新读了一遍当前主库，数据量已经比上一版又增加了一批。现在主库大约 233 MB，里面有 1246 个病毒条目，6687 条分离株或序列记录，22823 条蛋白记录，157 条宿主记录。数据库结构上还是 119 张表和 33 个分析视图。

感染和证据这块也有更新。现在感染或宿主关联记录是 3835 条，证据记录是 3243 条，参考文献是 322 篇。地理信息目前覆盖 48 个国家或地区。目标宿主总数现在是 123 个，比上一版增加比较明显，其中甲壳类目标宿主 88 个，软体动物目标宿主 21 个，其他水生无脊椎目标宿主 14 个。

目前的状态是，数据库已经不只是甲壳类病毒 metadata 表，而是一个可以围绕病毒、宿主、序列、地理、蛋白和文献证据一起检索和统计的系统。现在最核心的工作，是把这些新增记录继续做病毒学审校，尤其是宿主范围、感染证据、疾病证据和文献来源。

## 病毒与宿主

病毒内容这块，经典甲壳类病原仍然是数据库的主体。WSSV 相关记录最多，目前有 1298 条分离株或序列记录。TSV 有 492 条，IHHNV 有 147 条，YHV 有 105 条，罗氏沼虾诺达病毒有 83 条，OsHV-1 有 74 条。蟹相关环状病毒现在也比较靠前，有 113 条记录。

这个分布还是比较符合水产病毒学研究现状的：对虾病毒，特别是 WSSV，是数据最密集的一块；TSV、IHHNV、YHV、MrNV 这些经典病原也已经有比较清楚的位置。软体动物方向目前已经能看到 OsHV-1 和鲍疱疹病毒相关记录，但后续还要继续补宿主和疾病证据。

宿主方面，凡纳滨对虾仍然是记录最多的宿主，有 765 条；斑节对虾有 461 条；克氏原螯虾已经增加到 261 条；卤虫属有 182 条；棘冠海星有 130 条；软体动物未定类群现在有 126 条；罗氏沼虾有 100 条。这里面有些记录已经到物种，有些还只是“甲壳动物未定类群”或“软体动物未定类群”，后面需要继续细化。

从病毒科看，未分类病毒仍然很多，有 753 个条目。这主要反映宏病毒组数据带来的候选病毒很多。除此之外，Marnaviridae 有 153 个，Picornaviridae 有 140 个，Yanviridae 有 43 个，Solemoviridae 有 40 个，Nodaviridae、Malacoherpesviridae、Iflaviridae 等也已经进入库里。后面需要把经典病原和宏病毒组候选病毒分开解释，不能把所有检测到的病毒都直接当成致病病毒。

## 病毒学证据层

证据层目前也有变化。自然感染证据现在是 1052 条，比之前多；死亡率证据是 686 条，诊断证据 386 条，宿主范围证据 239 条，传播证据 174 条，致病性证据 169 条，暴发记录 152 条，毒力相关证据 125 条，温度相关证据 59 条。

这部分对后面写文章很关键，因为数据库不能只展示“有多少序列”，还要回答病毒学问题：这个病毒在哪些宿主里出现过？是自然感染还是共出现？有没有疾病暴发背景？有没有死亡率或病理表现？有没有传播证据？有没有温度或环境压力相关信息？

宿主关联方式现在分得更细。疾病暴发记录有 2276 条，宏病毒组共出现有 830 条，按宿主门类推断的有 360 条，来自 NCBI 注释的有 317 条，按名称推断的有 44 条，确认感染记录有 7 条。这个分层很重要，因为宏病毒组共出现只能说明候选关联，不能直接等同于致病；疾病暴发、自然感染、实验感染和病理证据的可信度更高。

## 扩展计划、对象和当前进展

扩展这块现在进展比较明显。目标宿主已经从上一版的 76 个增加到 123 个。甲壳类目标宿主现在是 88 个，软体动物目标宿主是 21 个，其他水生无脊椎目标宿主是 14 个。按宿主门类看，Arthropoda 有 111 条宿主记录，Mollusca 有 21 条，Echinodermata 有 7 条，Cnidaria 有 3 条，Porifera 有 1 条。

接下来的扩展还是按批次来。第一批是软体动物重点病原，主要补 OsHV-1、OsHV-1 microvariant、鲍疱疹病毒和 AVNV。第二批是多宿主或跨宿主病毒，主要是 CMNV 和 MDNV，这一块要特别标清楚不同宿主和传播证据。第三批是珊瑚和海绵病毒组，先作为病毒组候选记录导入，不直接写成致病病毒。第四批是棘皮动物，主要整理少量代表性病原和文献空白。

目前已经完成的是数据结构和第一批扩展入口。宿主表已经能记录门类和纲；病毒主表已经能记录宿主门类和发现方式；感染记录里也可以区分疾病暴发、确认感染、宏病毒组共出现、NCBI 注释和推断来源。软体动物和棘皮动物的数据已经开始进库，后面要把未定类群继续细化到更具体的宿主。

## 前端功能

前端这块目前可以支持常规展示。首页可以按病毒名、NCBI accession、病毒科属、宿主学名或中文名搜索。比如可以搜 WSSV、TSV、IHHNV，也可以搜凡纳滨对虾、斑节对虾、罗氏沼虾。

病毒详情页可以把一个病毒的分类、宿主记录、分离株、序列、蛋白、参考文献和证据放到一起看。统计页可以看主要病毒、主要宿主、基因组类型和证据覆盖情况。地图页可以看采样国家或地区，网络页可以看宿主-病毒关系，RdRp 和蛋白页面可以服务后续序列分析。

数据产物方面，已经有全序列 FASTA、完整基因组 FASTA、标准化元数据表、宿主-病毒网络表，以及多张 TSV 导出表。现场如果不跑后端，可以先用离线页面展示进展；如果要演示真实系统，再启动后端服务打开网页。

## 当前限制

现在主要限制有几个。第一，未分类病毒仍然很多，目前有 753 个条目。这不是坏事，说明宏病毒组带来了大量新候选，但这些记录需要和经典致病病毒分开解释。

第二，宿主和文献还要继续细化。现在有些记录仍然是 Unknown、甲壳动物未定类群、软体动物未定类群，后面要尽量细化到具体宿主或至少更准确的类群。参考文献里还有 48 条没有 PMID 或 DOI，需要补可追溯标识。

第三，宏病毒组候选记录很多。现在 discovery_context 里 metagenomic_survey 有 718 条，这些记录可以作为病毒多样性和宿主关联线索，但不能直接支撑致病性结论。后面需要用自然感染、疾病暴发、实验感染、死亡率和病理证据来给关键病毒分级。

第四，新导入的数据还要校对病毒、宿主、文献和序列之间的对应关系。尤其是新增的软体动物、棘皮动物和其他水生无脊椎记录，要确认不是宿主转录本、环境污染或非目标记录。

## 后续优化构建策略

后续构建我会分成数据层和系统层两部分推进。数据层的第一件事是做经典病原可信集。优先不是所有病毒平均用力，而是先把 WSSV、YHV、TSV、IHHNV、IMNV、MrNV、CMNV 这几个甲壳类经典病原整理扎实，包括宿主范围、地理分布、死亡率、诊断方法、传播证据和对应文献。

第二件事是宿主标准化。现在库里还有 Unknown、甲壳动物未定类群、软体动物未定类群这类记录。后面会尽量把它们细化到物种，至少也要细化到明确的宿主类群。这样后面做宿主谱和跨宿主分析时才不会混乱。

第三件事是文献证据追踪。现在数据库里已经有大量证据片段，但其中一部分还需要回到原文确认。后续会把证据分成自然感染、实验感染、疾病暴发、宏病毒组共出现、环境样本等不同等级。只有文献来源明确、能追到 PMID 或 DOI 的证据，才放进高可信结果里。

第四件事是清理非目标记录。比如疑似宿主转录本、mRNA、cDNA、EST、环境污染或者非目标宿主记录，这些不一定要全部删除，但要从发表集里剔除或单独标记，避免影响病毒学结论。

系统层主要是把前端和下载包做稳定。前端检索要支持病毒名、缩写、中文名、宿主名和 accession 的联合搜索；病毒详情页要更突出宿主范围、证据等级、文献来源、蛋白和地理分布；下载包要有稳定版本号、FASTA、元数据表、宿主-病毒网络和校验文件；同时保留导入脚本、数据来源、更新时间和质量检查报告，保证后面可以复现。

未来 3 到 4 周，主要整理前端和下载包。目标是首页搜索、病毒详情页、统计页、下载页都能稳定使用，并生成一个内部测试版本。

## 公共服务器部署流程

公共服务器部署我打算走“GitHub 管代码和版本，腾讯云负责域名和服务器”的路线。大致分七步。

第一步是版本冻结。先确定一个内部 release 版本，把主库、下载文件和前端代码冻结下来，形成 release_candidate 目录，同时确认哪些文件可以公开、哪些大文件需要单独归档或放下载目录。

第二步是整理 GitHub 仓库。把前端、后端、数据字典、部署说明、README、许可证和示例下载文件放到 GitHub；大型 SQLite 主库、FASTA 和备份文件不直接塞进代码仓库，而是放在服务器下载目录或后续 DOI 数据仓库里。

第三步是在腾讯云购买域名和服务器。服务器可以先选一台轻量应用服务器或 CVM，系统用 Linux。域名买好以后做 DNS 解析。如果后续需要长期公开访问，还要按腾讯云流程处理备案。

第四步是在服务器上部署运行环境。可以先用 Python 虚拟环境直接部署，后面稳定后再考虑 Docker。核心是配置 FastAPI 后端、Jinja 前端模板、静态文件、SQLite 数据库路径和环境变量。

第五步是配置公网入口。用 Nginx 做反向代理，后端用 Uvicorn 或 Gunicorn 跑起来，域名解析到服务器，再配置 HTTPS 证书，同时保证大文件下载和静态资源能正常访问。

第六步是同步数据和下载文件。把 SQLite 主库、FASTA、Excel、TSV、宿主-病毒网络文件和校验文件上传到腾讯云服务器，保证网页查询和下载页对应同一个版本。

第七步是测试、监控和长期归档。上线前测试首页、搜索、病毒详情页、统计页、下载页和 API；上线后保留访问日志、错误日志、数据库备份和异常提醒。等版本稳定后，再准备 Zenodo 或 Figshare DOI，写清楚引用格式、数据许可和版本号。

## 下一步计划

下一步分两条线。第一条线是甲壳类经典病原高可信整理。优先整理 WSSV、YHV、TSV、IHHNV、IMNV、MrNV、CMNV，把宿主范围、地理分布、死亡率、诊断方法、传播证据和文献来源补齐。

第二条线是水生无脊椎扩展。软体动物优先，先把 OsHV-1、OsHV-1 microvariant、鲍疱疹病毒、AVNV、CMNV、MDNV 相关记录补齐。然后按批次加入珊瑚病毒组、海绵病毒组和棘皮动物少量病原记录。每扩展一个类群，都同步补宿主标准名、病毒发现方式、文献来源和证据等级。

短期具体工作就是三件事：一是校对新增记录和宿主、文献、序列之间的对应关系；二是补齐重点病毒的 PMID、DOI 和原文证据；三是把明显不是病毒基因组的记录清理掉。等这些完成后，再考虑公网部署、数据 DOI、GitHub 公开和数据库论文。

这个项目现在最有价值的不是单纯收集了多少条序列，而是开始把水产病毒学里分散的病毒、宿主、疾病、地理和文献证据连起来。下一阶段要做的，是把这些连接从“候选整理”变成“病毒学上经得起追问的证据链”。
